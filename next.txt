### segment management
need manifest to store the latest sst id.
- manifest
-- stores latest segment id

- segments_mgr
-- scans 'segments/' topology
-- caches topology in memory (?)

- engine
-- uses manifest to get path to the next latest segment
-- iterates over the segment to find the key

then need sstable_reader to use segments_mgr to get the path to the appropriate sstable.
for example, on get path, need to get path to the latest segment in level_0, then next and so on until level_0 is exhausted, then need the same logic for other levels. these paths can be cached on the startup.
compaction logic will effect segment distrubtion of levels, so in future it should update segments_mgr/manifest.
